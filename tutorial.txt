
On commence par charger le code lisp :

 M-x load-file /path/to/rfcinfo.el

ou si on l'a installé dans un répertoire cherché par emacs (eg en
plaçant (add-to-list 'load-path "<mon répertoire>") dans le .emacs),
avec un

 M-x load-library rfcinfo

Il affiche un message d'erreur.
La première chose à faire est de télécharger le fichier rfc-index et
construire la base (cela prend quelques secondes) :

 M-x rfcinfo-refresh

Maintenant on peut invoquer la fonction rfcinfo-show, soit par M-x
rfcinfo-show, soit par C-c r.  On entre alors le numéro de RFC choisi,
par exemple 2205.

Joli ?
Les flèches haut et bas permettent de se déplacer sur les numéros de
RFC, la flèche droite de 'suivre' le RFC sur lequel est le curseur, la
gauche de revenir à l'état précédent. 'q' permet de fermer la fenêtre.

C-c r ne demande pas de numéro si le curseur est présent sur un numéro
de rfc, comme rfc1034, 1035 (essayer ici !)

La deuxième fonctionalité est rfcinfo-open, accessible par C-c R.  On
entre le numéro de RFC, il est téléchargé et une copie est gardée dans
le répertoire désigné par la variable rfcinfo-cache-dir (par défaut,
~/RFC/Cache).  S'assurer que ce répertoire existe bien !

L'affichage est fait avec rfcview si il est présent.  Si ce n'est pas
le cas, on peut le télécharger à
http://www.loveshack.ukfsn.org/emacs/rfcview.el
(pour Debian il est dans le paquetage emacs-goodies-el)

rfcinfo-open peut aussi ouvrir le RFC à un emplacement déterminé, ce
qui est très pratique pour garder une référence sur un point précis
dans une doc.  Par exemple (essayer C-c R sur les références aux RFC,
puis 'q' pour quitter) :

---
rfc2181-5    RR has label, class, type and data. [+ defining RRSet]
rfc2181-5.2  Resource Records also have a time to live (TTL).


AA bit: the data matching the question name is authoritative
(rfc1034-6.2.7) 

rfc1035-4.1.1+42 (ligne 42 après la section 4.1.1)
---

Et voilà.  Voir les commentaires du fichier rfcinfo.el pour plus de
détails.

Tout commentaire est bienvenu.
